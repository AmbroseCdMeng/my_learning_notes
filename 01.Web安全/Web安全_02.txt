第二章 Web安全的乌云
		———— SQL 注入

	SQL 注入 作为网络安全危害的榜首。
	其通过构建特殊的输入作为参数传入Web应用程序。
	
	特点：
		直接、暴力
	漏洞原因：
		主要原因是程序对用户输入的数据过滤不够细致，致使非法数据入侵
	
	SQL 注入 主要分平台层注入 和 代码层注入
		1、平台层注入
			由不安全的数据库配置或数据库平台的漏洞所导致
		2、代码层注入
			由于程序员对输入未进行细致的过滤，从而导致非法数据的执行
	产生的原因主要表现在：
		1、不当的类型处理
		2、不安全的数据库配置
		3、不合理的查询集处理
		4、不当的错误处理
		5、转义字符的处理不合适
		6、多个提交处理不当

	常见的SQL注入工具
		1、啊D
		2、名小子
		3、sqlmap  --Python 2.7环境
		★ 工具基本所有的都注入后门，不建议使用

	常见注入手段
		1、Get 注入
		2、Post 注入
		3、cookie 注入
		4、旁站注入
		5、二次注入
		6、其他奇淫技巧
